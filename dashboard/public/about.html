About This Branch

Crucix 中文增强分支说明

这是一个基于上游 Crucix 的非官方本地化增强版本。目标不是冒充官方，而是在保留原项目核心结构的前提下，把它调整成更适合中文用户直接观察、排查和使用的本地情报终端。
上游仓库 返回仪表盘

上游信息

官方上游仓库：calesthio/Crucix

本仓库不是官方发行版，也不代表上游维护者的正式配置、文档或默认路线。

这个分支的目标

  • 降低中文用户上手门槛
  • 补齐国内新闻和财讯参考面
  • 增强时间排序、分页、数据流展示和机翻入口
  • 在不破坏主框架的前提下提高可读性和可用性

当前增强点

  • 简体中文界面和来源中文映射
  • 中国新闻网、人民网、新华网、央视网、新浪新闻聚合
  • orz-ai/hot_news 财讯热榜接入
  • 同题聚类、冲突提示、自然滚动快讯流和中国新闻数据流图
  • 中港指数、中国主题 ETF 与可选航空延误观察
  • China Finance RSS 桥接：东方财富/同花顺 7×24 快讯接入
  • Microsoft 365 Copilot / Azure OpenAI 集成，支持 AI 增强观点

使用建议

如果你需要尽量接近官方默认行为，请直接使用上游仓库。如果你需要中文界面、国内新闻、机翻入口和更贴合中文环境的展示，可以使用当前分支。

集成与数据源

China Finance RSS Bridge

轻量级 RSS 桥接服务，将财联社电报、东方财富 7×24、同花顺 7×24、雪球用户动态转为标准 RSS 2.0。单文件 Python，最小依赖。

  • GitHub：yuxuan-made/china-finance-rss
  • 掘金：写了个单文件 Python 脚本，把财联社/东财/同花顺/雪球转成 RSS

在 .env 中设置 CN_FINANCE_RSS_URL 指向运行中的 RSS 桥接服务器即可启用（默认 http://localhost:8053）。

Microsoft 365 Copilot API

通过 Azure OpenAI 或 Microsoft Graph Copilot Chat API (preview) 为可执行观点面板提供 AI 增强分析。

  • 文档：Microsoft 365 Copilot APIs 概述
  • 需 M365 Copilot 许可证或 Azure OpenAI 部署

在 .env 中设置 LLM_PROVIDER=copilot 并配置 Azure OpenAI 或 Graph 凭据。

实现方向

这个分支的主要改动集中在 dashboard/inject.mjs 和 dashboard/public/jarvis.html 两层：

  • 后端聚合层负责把新增的中国新闻、热榜、航空延误等来源统一成前端可消费结构。
  • 前端页面负责把这些数据渲染成分页卡片、中国新闻聚合面板、机翻入口和轻量数据流图。
  • China Finance RSS 桥接层将东方财富/同花顺的 7×24 快讯通过本地 RSS 服务接入，与现有 orz.ai 热榜互补。
  • LLM 层新增 Microsoft Copilot 提供商，支持 Azure OpenAI 部署和 M365 Graph Chat API 两种模式。
